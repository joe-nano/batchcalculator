PDF/TeX reports

In the latest batch calculator release there is a built in pdf report generator based on reportlab package that will generate a report wihtout using eny external programs. The pdf can be generated for the conventional batch calculation (moles to masses) or from the composition calculation (masses to moles).

An alternative and more flexible option is generating a report in tex format which can be edited further or used to generate the pdf file. To generate the pdf a tex distribution is required. Although a TeX distribution is not a formal dependency and the program will work without any trouble without it, it is required if you want to produce nice pdf reports. If you don't know what TeX is, but you would like to know more TUG (TeX Users Group) website is a good place to start.

If you just want to get it to work here are some quick links to major TeX distributions for different platforms

  • TeX Live for Unix/Linux
  • MacTeX for Mac OS
  • MikTeX for Windows

Installation on Windows and Mac

For windows and MacOSX GUI installers are available for each release that can be downloaded from here. After downloading adn double clicking on the installer a familiar wizard should appear and you only need to follow instructions on the screen.

Database

The program uses an SQLite3 database to store all of the data on chemicals and all of the details used in the calculations. The database distributed with the code is limited but can easily be extended with additional reactants. The schema of the database in shown on the diagram below.

Statring from version 0.2.0 adding new records or modifying and deleting existing ones is possible from within the batch calculator. This new functionality makes any external database browser redundant.

The program offers an option to use an external database (e.g. from other users) or creting a new customized database from scratch.

Installation from source

Prerequisites

Installation from source is the only choice for now for linux users but it is also an option for other users if they want to develop the package or test some new features.

Before installing batch calculator on linux you should make sure you have wxPython installed which is the GUI toolkit used. wxPython is a multiplatform GUI toolkit and its latest version together with detailed instructions on how to install it are on wxPython's webpage.

Below in the list of all the packages that batch calculator depends on:

  • Python 2.7.x,
  • wxPython run and tested with wx version 2.8.12.1,
  • numpytested with version 1.8.1,
  • SQLAlchemy 0.9.7,
  • Jinja2 2.7.3,
  • ObjectListView 1.2,
  • ReportLab 3.1.8,

Manual installation

To get the program you can either download the latest stable release from the repository or clone the whole repository to have the very latest version {% highlight bash %} git clone https://github.com/lmmentel/batchcalculator.git {% endhighlight %} If you installed all the prerequsites donwloaded or clone the repository and unpacked then to finally install the package cd to the package direcotry and type {% highlight bash %} $ [sudo] python setup.py install {% endhighlight %} If the installation finishes without errors you should be able to start the GUI from the command line by typing {% highlight bash %} zbc.py {% endhighlight %}
